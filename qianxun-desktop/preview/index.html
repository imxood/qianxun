把 Chat 放回主工作区

千寻不再是 VSCode IDE 也不是 Kanban 看板. 它是一个 以 Chat 为中心 的工作台: 项目是有记忆的容器, 会话是一次次主题对话, 主 Agent 在需要时拉一群子 Agent 干活 (Plan), 子 Agent 上下文持久化可点入, 项目经验跟会话纪要自动沉淀.

关键交互规则 (从 mavis-team 借来)

  • 1 个 PlanTask = 1 个可验证交付物 (跟 mavis-team SKILL.md:217 一致)
  • verifier 独立 re-derive, 不读 producer diff 盖章
  • 单任务 30 分钟硬上限, 整体 30 分钟上限
  • skip verify 必须有 user-written reason (结构层硬约束)
  • 子会话终止后只读, 追问回主会话
