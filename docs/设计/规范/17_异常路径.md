# 14 缺口的异常路径

> 状态: 草稿 (待 code review) | 适用范围: 14 缺口各自的失败场景 | 最后更新: 2026-06-11 | 版本: v0.1

## 1. 目的

14 缺口文档大部分只写 happy path。**生产级必须考虑边界**:
- 网络/DB/IO 失败
- LLM 输出异常
- 并发冲突
- 资源耗尽

本文档列出 14 缺口的关键异常路径 + 期望处理。

## 2. 异常路径分类

| 类别 | 处理原则 |
|---|---|
| **可恢复 (Retry)** | 自动重试 + 上报 SseEvent, 用户不感知 |
| **可降级 (Fallback)** | 走次优路径, 给用户 WARN 提示 |
| **不可恢复 (Abort)** | 返回明确错误, UI 显示, 不静默吞 |
| **数据一致性 (Rollback)** | 事务回滚, 状态不变 |

## 3. 逐缺口异常路径

### 缺口 01: Hook 退出码 + 熔断

| 异常 | 处理 | 类别 |
|---|---|---|
| 单次 hook panic | `HookResult::Error("panic: msg")` | 可恢复 |
| 单次 hook 超时 | `HookResult::Error("timeout 5s")` | 可恢复 |
| 同 hook 连续 3 次 Error | circuit open, 60s 内禁用, 发 `HookDisabled` SseEvent | 可降级 |
| 60s 后自动恢复, 仍失败 | 再次 circuit open | 可降级 |
| **所有 hook 都熔断** | 主循环 fallback 到无 hook 模式, UI 提示 | 不可恢复 |

### 缺口 02: LLM 错误分类

| 异常 | LlmErrorKind | RecoveryAction |
|---|---|---|
| 401 unauthorized | `Auth` | refresh → 再 401 → `AuthPermanent` → `Abort` |
| 403 forbidden | `AuthPermanent` | `Abort` |
| 402 billing | `Billing` | `RotateProvider` |
| 429 rate limit | `RateLimit` | `Retry { delay: 5s }` |
| 503 overloaded | `Overloaded` | `Retry { delay: 10s }` |
| 500 server error | `ServerError` | `Retry { delay: 1s }` (3 次) |
| 网络超时 | `Timeout` | `Retry { delay: 2s }` |
| ContextOverflow | `ContextOverflow` | `CompressContext` |
| PayloadTooLarge (413) | `PayloadTooLarge` | `CompressContext` |
| ModelNotFound (404) | `ModelNotFound` | `FallbackModel("next")` |
| ContentPolicyBlocked | `ContentPolicyBlocked` | `Abort` (per-request 确定性, 不重试) |
| InvalidThinkingSig (Anthropic) | `InvalidThinkingSig` | strip thinking block + `Retry` |
| 全部 provider 失败 (缺口 12) | `AllProvidersFailed` | `Abort(msg)` 给 UI |

### 缺口 03: SubAgent 工具白名单

| 异常 | 处理 | 类别 |
|---|---|---|
| sub-agent 调非白名单工具 | `ToolError::Denied`, 发 `ToolDenied` SseEvent | 不可恢复 (单次拒绝) |
| sub-agent 递归 fork_subagent | 同上 | 不可恢复 |
| sub-agent 写主 memory | 同上 | 不可恢复 |
| sub-agent 调 send_message | 同上 | 不可恢复 |
| **白名单配置错误** (tool_filter: ["nonexistent"]) | 启动时校验, 报错退出 | 配置错误 |

### 缺口 04: Skill 生命周期

| 异常 | 处理 | 类别 |
|---|---|---|
| skill 文件 YAML 解析失败 | 跳过该 skill, tracing WARN | 可降级 |
| tick 跑时 DB lock | retry 3 次, 仍失败 WARN 不阻塞 | 可恢复 |
| 30 天归档时文件被外部改 | 跳过归档, 记录差异 | 可降级 |
| 同一 skill 多个 Candidate 冲突 | 取 last_used_at 最新 | 数据一致性 |

### 缺口 05: 后台异步任务

| 异常 | 处理 | 类别 |
|---|---|---|
| 任务 spawn 失败 (OOM / process limit) | `TaskStatus::Error("spawn failed: ...")`, 发 `TaskFailed` SseEvent | 不可恢复 (单任务) |
| 子 task (sub-agent) panic | 同上 | 不可恢复 |
| 用户 cancel 时任务正在写 DB | cancel 后 rollback (事务回滚) | 数据一致性 |
| FIFO 队列满 (默认 100) | 拒绝新任务, 返 `QueueFull` 错误给 UI | 不可恢复 |
| 5 并发全部失败 | 用户可手动 retry, 不自动重启 | 不可恢复 |
| 任务结果 > 10MB | 截断到 10MB, tracing WARN | 可降级 |

### 缺口 06: 压缩前 Memory Flush

| 异常 | 处理 | 类别 |
|---|---|---|
| LLM 提取 durable info 失败 | 走空 Vec, 不 flush, 仍 compact | 可降级 |
| flush 后 memory.save() 失败 | 1 次 retry, 仍失败则 tracing ERROR, 不阻塞 compact | 可恢复 |
| compact 本身失败 | **不返回错误**, 返回原 conversation (无压缩) | 可降级 |
| soft_threshold (0.75) 触发 flush 但没东西可 flush | noop, 不记 error | 正常 |

### 缺口 13: 双层循环

| 异常 | 处理 | 类别 |
|---|---|---|
| inner_loop panic | outer_loop 捕获, 发 `InnerFailed` SseEvent | 不可恢复 (单 turn) |
| steer 消息格式错误 | 忽略该 steer, tracing WARN | 可降级 |
| user_input channel 关闭 (UI 断开) | inner_loop 自然退出, session 进入 Idle | 正常 |
| budget 用完 | `InnerEvent::BudgetExhausted`, inner_loop break | 正常 |
| 事件订阅者掉线 | `subscribe_events` Receiver 关闭, 但 inner 仍跑 | 正常 |

### 缺口 12: Provider 三层 Failover

| 异常 | 处理 | 类别 |
|---|---|---|
| Layer 1 retry 3 次仍失败 | 走 Layer 2 (ProviderChain) | 可降级 |
| Layer 2 链上所有 provider 都 fail | 走 Layer 3 (AdaptiveRouter hedge) | 可降级 |
| Layer 3 hedge 全部超时 | `LlmErrorKind::AllProvidersFailed` → 返 `Abort` | 不可恢复 |
| circuit breaker 阈值 (5 次) 触发 | 60s 内不开新请求到该 provider | 可降级 |
| scoreboard 数据竞争 | 用 RwLock, 不锁死 | 正常 |

### 缺口 13: Hook 五层 Tier

| 异常 | 处理 | 类别 |
|---|---|---|
| 同 tier 多个 hook 中间 Block | 立即返回 Block, 后面的不触发 | 正常 |
| hook tier 配置缺失 (e.g. Transform tier 无 hook) | 跳过该 tier, 不报错 | 正常 |
| 同 hook 注册到多 tier | 警告 + 拒绝注册 | 配置错误 |

### 缺口 12: Hashline Edit

| 异常 | 处理 | 类别 |
|---|---|---|
| 提交时 hash 不匹配 (stale) | `ToolError::StaleEdit { line, expected_hash, current_hash, current_content }` | 不可恢复 (单次拒绝) |
| LLM 收到 StaleEdit 重试 | 用新 hash 再 edit, 成功 | 可恢复 |
| hash 字符表配置错误 | 启动时校验, 默认 16 字符 | 配置错误 |
| binary 文件尝试 hashline | 拒绝, 返 `ToolError::Unsupported` | 不可恢复 |

### 缺口 11: Verdict + BDD

| 异常 | Verdict | 处理 |
|---|---|---|
| cargo test 全 pass | `Pass` | 继续 |
| cargo test 部分 fail | `Uncertain` | 阻断 turn, UI 提示用户确认 |
| cargo test 全 fail | `Fail` | `HookResult::Modify(retry_args)` 自动重试 |
| cargo 不存在 | `Skip` | 不强验, 继续 (≠ Pass) |
| test_selector 跑超时 (10min) | `Uncertain` | 同上 |
| boundary glob 无文件改动 | `Uncertain` (reason: "未触及目标文件") | 同上 |

### 缺口 13: Context Window 五层

| 异常 | 处理 | 类别 |
|---|---|---|
| 5 层都查不到 (新 model) | fallback 到 `128_000` 默认 | 可降级 |
| API metadata 抓取失败 | 跳过该层, 走 heuristic | 可降级 |
| API metadata 缓存过期 | 启动时刷新, 24h 内不重抓 | 正常 |
| model 名拼错 | heuristic 返回 None → fallback default | 可降级 |

### 缺口 14: Knowledge 五状态

| 异常 | 处理 | 类别 |
|---|---|---|
| promote 时 verification_refs 为空 | `GateReport.allowed = false`, 返 `Err` | 不可恢复 (单次拒绝) |
| gate 拒绝, 已是 Promoted 状态 | 状态不变 (不允许 demote 跳过 gate) | 数据一致性 |
| 30 天未访问自动归档时 DB lock | retry 3 次, 失败 WARN | 可恢复 |
| 用户显式 confirm 时 DB 失败 | rollback, UI 提示重试 | 数据一致性 |

### 缺口 12: Session Queue 五种模式

| 异常 | 处理 | 类别 |
|---|---|---|
| SwitchMode 时 queue 还有未处理消息 | 先 drain 旧, 再切新 mode | 数据一致性 |
| Speculative 预跑失败 | 静默丢弃, 不影响当前 turn | 可降级 |
| Interrupt 时正在写 conversation | 写入完成的 message 保留, 未启动的不写 | 数据一致性 |
| Collect mode 2s 内用户连发 100 个 | 取前 5 个, 剩余入下批 | 正常 |

## 4. 跨缺口的"全部失败"边界

| 场景 | 行为 |
|---|---|
| **所有 hook 全熔断** | 主循环 fallback 无 hook, UI 提示 |
| **所有 provider 全失败** | 返 `Abort("所有 provider 都不可用, 请稍后重试")`, 不静默 |
| **所有 background task 都 fail** | 用户可手动 retry |
| **memory 写盘全失败** | tracing ERROR + session 标 ReadOnly, 不写新数据 |
| **Tauri IPC 全断** | 前端 store 缓存最后状态, 重连后 sync |

## 5. 不在本设计范围

- 不实现具体 retry / fallback 代码 (各缺口文档已写 happy path, 实施时按本文档补 try/catch)
- 不做自动混沌测试 (后续工作项)
- 不做 SLA / 错误率告警 (后续工作项)
