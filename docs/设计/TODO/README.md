## 命名规范

每个工作项一个子文件夹, 命名 `YYYY-MM-DD_事项名/`:

```
TODO/
└── 2026-06-11_v2_缺口补齐_14项/
    ├── README.md     ← 工作项入口 (目标/范围/状态/关联/下一步)
    ├── TODO.md       ← 14 缺口步骤 + 完成标准
    ├── 记录.md       ← 调研过程 + 借鉴源对比
    └── 验收.md       ← 总体验收 + 单缺口引用
```

**文件职责** (跟 document_rules.md 一致):
- `README.md`: 入口, 1 段目标 + 范围 + 状态表 + 关联文档 + 下一步
- `TODO.md`: 可执行任务清单, 状态, 阻塞
- `记录.md`: 执行过程, 关键判断, 中间结论, 验收证据
- `验收.md`: 验收检查表 + 单缺口引用
