## 4. 改动原则 (避免大文件)

1. **新概念 → 新文件** (tier.rs, failover.rs, capabilities.rs, hashline.rs, dual.rs, event.rs, lifecycle.rs, verify/, knowledge.rs, gate.rs, background_task.rs, queue.rs)
2. **改概念 → 改原文件**, 不重命名 (`registry.rs`, `handler.rs`, `compact.rs`, `mod.rs` 原地改)
3. **模块依赖单向**: qianxun-core ← qianxun-runtime ← qianxun-desktop (无反向依赖)
4. **不引新 crate**: 14 缺口只复用现有依赖 (tokio / serde / dashmap / rusqlite)
5. **测试就近**: 新模块测试就近放 (mod tests), 不集中到 tests/

## 5. 不在本设计范围

- Tauri UI 重设计 (缺口 05 的 TaskListPanel 只占位, 后续 UI PR 详细)
- SQLite schema 迁移脚本 (留 v3.0 release)
- 跨 workspace 共享类型抽取 (千寻是 monorepo, 各 crate 独立)
