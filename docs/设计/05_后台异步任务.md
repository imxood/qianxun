# 缺口 05: 后台异步任务

> 状态: 草稿 (待 code review) | 适用范围: qianxun-core / qianxun-runtime | 最后更新: 2026-06-10 | 版本: v0.1

## 借鉴源

[oh-my-opencode](E:\git\ai\oh-my-opencode) `background-task` + `background-continuation` + 5-class 调度 + FIFO 队列。

## 问题

千寻 v2 `processing_loop_v2` 是**同步循环**: 用户发 prompt, 等待整个 loop 跑完才能看下一步结果。

后果:
- 长任务 (10 分钟研究) 用户不能切走
- 切走后回来结果丢了
- 多个长任务不能并行

## 方案

### 5.1 5 状态任务

```rust
// qianxun-runtime/src/background_task.rs (新)

pub enum TaskStatus {
    Queued,     // FIFO 队列里
    Running,    // 已起 tokio task
    Done,       // 完成, 结果可查
    Error(String),  // 失败
    Cancelled,  // 用户取消
}

pub struct BackgroundTask {
    pub id: TaskId,                // bg_<ts>_<rand>
    pub parent_session_id: SessionId,
    pub description: String,       // 用户可见: "调研 RAG in Rust"
    pub status: TaskStatus,
    pub started_at: Option<Instant>,
    pub finished_at: Option<Instant>,
    pub result: Option<String>,    // LLM 最终输出
    pub progress: TaskProgress,    // 步数 / token / 当前 sub-agent
}
```

### 5.2 BackgroundTaskManager

```rust
pub struct BackgroundTaskManager {
    tasks: Arc<RwLock<HashMap<TaskId, Arc<RwLock<BackgroundTask>>>>>,
    sem: Arc<Semaphore>,           // max 5 并发
    queue: mpsc::UnboundedSender<TaskId>,  // FIFO
}

impl BackgroundTaskManager {
    /// 5 并发 per session, FIFO 排队
    pub async fn start(&self, parent: SessionId, description: String, spec: SubAgentSpec) -> TaskId;
    pub async fn status(&self, id: TaskId) -> TaskStatus;
    pub async fn cancel(&self, id: TaskId);
    pub async fn resume(&self, id: TaskId, prompt: String) -> TaskId;  // 续接
    pub async fn list(&self, parent: SessionId) -> Vec<TaskInfo>;
}
```

### 5.3 SseEvent 加 4 变体

```rust
// qianxun-runtime/src/sse.rs

TaskQueued { task_id, position },          // 排队中
TaskStarted { task_id, description },      // 开始跑
TaskProgress { task_id, step, total },     // 进度更新 (每 5 步)
TaskCompleted { task_id, result },         // 完成
TaskFailed { task_id, error },             // 失败
```

### 5.4 RuntimeApi 加 4 方法

```rust
// qianxun-runtime/src/api/trait_def.rs

async fn start_background_task(&self, parent: &str, spec: SubAgentSpec) -> TaskId;

```

### 5.5 Tauri UI 展示

Tauri 新增组件 `TaskListPanel.svelte`:
- 列出所有任务 (按状态分组: Running / Queued / Done)
- 点 Done 任务 → 弹结果模态框
- 点 Running 任务 → 弹进度条
- 点任何任务 → 按钮 "Resume" 续接

## 不做什么

- 不做跨 session 任务共享 — 任务 = sub-agent, parent 隔离
- 不做任务调度优先级 — 简单 FIFO
- 不做任务结果持久化到 SQLite — Done 后 24h 删, 减少存储

## 验收

- [ ] 用户发 6 个长任务 → 前 5 个 Running, 第 6 个 Queued
- [ ] 任务 1 跑完 → 任务 6 自动开始
- [ ] 用户切走 5 分钟 → 切回, 任务仍在跑, 进度正常
- [ ] cancel 任务 → status=Cancelled + SseEvent
- [ ] resume 任务 → 新 task_id (不覆盖旧结果)
- [ ] 5 状态流转图: Queued → Running → Done/Error/Cancelled
