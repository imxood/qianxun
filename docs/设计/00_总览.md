> 状态: 草稿 (待 code review) | 适用范围: qianxun-core / qianxun-runtime | 最后更新: 2026-06-10 | 版本: v0.1

> **本文件是 14 缺口的实施总账** (清单 + 排期 + 借鉴源)。架构总览见 [总设计.md](./总设计.md)。

## 1. 为什么有这 14 个

[./agent_loop_v2.md](./agent_loop_v2.md) 完成后, 千寻从"能跑 LLM 循环"升级到"具备 4 模式 + 双轴权限", 但跟 opencode/oh-my-opencode/Claude Code 等生产级 Agent 仍有显著差距。

扫描了 `E:\git\ai\` 26 个项目后, 提炼出 14 个**千寻 v2 还没的**核心能力, 分两类:

| 优先级 | 数量 | 总行数 | 阶段 |
|---|---|---|---|
| 🔴 P0 (核心缺陷) | 5 | ~830 | v2 必备, 无它们 v2 不完整 |
| 🟡 P1 (重要增强) | 9 | ~1330 | v2+1 提升到生产级, 按 ROI 选做 |

## 3. 实施顺序 (按依赖 + ROI, 按新编号)

```text
Phase 1 (L2 P0 必做, 5 项)
  缺口 02 (LLM 错误分类) ← 无锚点, 立即
  缺口 01 (Hook 退出码)   ← 跟 v2 E1 同步
  缺口 04 (Skill 自学习)  ← 无锚点, 立即
  缺口 03 (SubAgent 白名单) ← 跟 v2 E4 同步
  缺口 05 (后台异步任务)  ← 跟 v2 E4 同步

Phase 2 (L3 P1 ROI 高, 5 项)
  缺口 06 (Memory Flush)      ← 无锚点, 立即
  缺口 07 (Hook 五层 Tier)    ← 无锚点, 立即
  缺口 08 (Hashline Edit)     ← 无锚点, 立即
  缺口 09 (Context Window)    ← 无锚点, 立即
  缺口 10 (Queue 五种模式)    ← 跟 v2 E2 同步

Phase 3 (L4 P1 ROI 中, 2 项)
  缺口 11 (Verdict + BDD)     ← 跟 v2 E3 reflect 联动
  缺口 12 (Provider Failover) ← 依赖 02

Phase 4 (L5 P1 投资大, 2 项)
  缺口 13 (双层循环)          ← 跟 v2 E2 同步, 重构
  缺口 14 (Knowledge Gate)    ← 跟 06 联动
```

## 5. 总行数估算

- v2 当前: ~2180 行
- 14 缺口: ~2160 行
- **合计 v2+1: ~4340 行**

**对比参考**:
- oh-my-opencode: 62 hooks + 11 agents + 5-tier = ~6000 行 (3 个 dev 团队 6 月工作量)
- 千寻 v2+1 4340 行 = 跟单 dev 3-4 月工作量, **单 AI agent 可分阶段 1-2 月实施完毕**
