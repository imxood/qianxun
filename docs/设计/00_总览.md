# 14 个设计缺口总览

> 状态: 草稿 (待 code review) | 适用范围: qianxun-core / qianxun-runtime | 最后更新: 2026-06-10 | 版本: v0.1

## 1. 为什么有这 14 个

[agent_loop_v2.md](../10_事实源/架构/agent_loop_v2.md) 完成后, 千寻从"能跑 LLM 循环"升级到"具备 4 模式 + 双轴权限", 但跟 opencode/oh-my-opencode/Claude Code 等生产级 Agent 仍有显著差距。

扫描了 `E:\git\ai\` 26 个项目后, 提炼出 14 个**千寻 v2 还没的**核心能力, 分两类:

| 优先级 | 数量 | 总行数 | 阶段 |
|---|---|---|---|
| 🔴 P0 (核心缺陷) | 5 | ~830 | v2 必备, 无它们 v2 不完整 |
| 🟡 P1 (重要增强) | 9 | ~1330 | v2+1 提升到生产级, 按 ROI 选做 |

## 2. 14 缺口一览

| # | 名称 | 借鉴源 | 价值主张 |
|---|---|---|---|
| 01 | Hook 退出码 + 熔断 | octos | 防 hook 故障雪崩 |
| 02 | LLM 错误分类 | hermes-agent | 22 种错误 → 正确恢复动作 |
| 03 | SubAgent 工具白名单 | microclaw | 防 sub-agent 递归/越权 |
| 04 | Skill 自学习 | opencrust | skill 长期健康 (5+ 次蒸馏 + 30 天归档) |
| 05 | 后台异步任务 | oh-my-opencode | 长任务不卡 UI + 用户回来查结果 |
| 06 | 压缩前 Memory Flush | openclaw-mini | 长任务关键信息不丢 |
| 07 | 双层循环 | openclaw-mini | 用户中途输入不冲突 |
| 08 | Provider 三层 Failover | octos | 99.9% 可用性 |
| 09 | Hook 五层 Tier | oh-my-opencode | 大规模 hook 不乱 |
| 10 | Hashline Edit | oh-my-opencode | 写文件正确性 (防 stale) |
| 11 | Verdict 四态 + BDD | agent-spec | in-loop 自验, 减少人工验收 |
| 12 | Context Window 五层优先 | moltis | 正确 compaction, 不浪费 token |
| 13 | Knowledge 五状态 + Gate | mempal | memory 质量 (验证后 promote) |
| 14 | Session Queue 五种模式 | octos | 用户交互灵活 |

## 3. 实施顺序 (按依赖 + ROI)

```text
Phase 1 (P0, v2 必备, 必做)
  缺口 01 (Hook 退出码) ← 跟 v2 E1 同步
  缺口 02 (LLM 错误分类) ← 跟 v2 E1 同步
  缺口 03 (SubAgent 白名单) ← 跟 v2 E4 同步
  缺口 04 (Skill 自学习) ← 独立, 后置
  缺口 05 (后台异步任务) ← 跟 v2 E4 同步

Phase 2 (P1, v2+1 选做, 按 ROI)
  缺口 09 (Hook 五层 Tier) ← 低垂果实, 优先
  缺口 06 (Memory Flush) ← 低垂果实, 优先
  缺口 12 (Context Window 五层) ← 中等 ROI
  缺口 10 (Hashline Edit) ← 中等 ROI
  缺口 14 (Queue 五种模式) ← 中等 ROI
  缺口 11 (Verdict + BDD) ← 跟 Reflect 联动
  缺口 07 (双层循环) ← 重构, 后置
  缺口 13 (Knowledge Gate) ← 跟 Memory Flush 联动
  缺口 08 (Provider Failover) ← 投资大, 最后
```

## 5. 总行数估算

- v2 当前: ~2180 行
- 14 缺口: ~2160 行
- **合计 v2+1: ~4340 行**

**对比参考**:
- oh-my-opencode: 62 hooks + 11 agents + 5-tier = ~6000 行 (3 个 dev 团队 6 月工作量)
- 千寻 v2+1 4340 行 = 跟单 dev 3-4 月工作量, **单 AI agent 可分阶段 1-2 月实施完毕**
