## 1. 角色与节奏

| 角色 | 模式 | 工具 |
|---|---|---|
| 执行人 | AI agent (单 AI 全程) | Claude / 千寻自身 |
| 节奏 | 每缺口 1 完整会话 (设计→实现→测试→验收) | 14 个独立会话 |
| 文档驱动 | 14 缺口文档是 spec, AI 读完直接编码 | docs/设计/01-14_*.md |
| 验证 | 跑测试 + 同步文档 + commit | cargo test / git |

**约束**:
- 不批量改 14 个缺口, **每缺口 1 commit**, 方便回退

- 跨缺口集成 (4 个集成点) 单独 PR, 不混在主缺口

## 5. AI agent 实施模板 (单缺口流程)

```bash
# 步骤 1: 读 spec
Read docs/设计/NN_缺口名.md
# (15 分钟理解)

# 步骤 2: 写实现 + 单元测试
# 涉及文件: 主设计.md §2 落点表
Write qianxun-core/src/...
Write qianxun-core/src/.../tests.rs

# 步骤 3: 跑测试
cd qianxun && cargo test -- <缺口相关模块>
# 期望: 248+ 旧测试全过 + 3+ 新测试过

# 步骤 4: 同步文档
Read 10_事实源/runtime-state.md  # 找 API 表
Edit 加新方法到 API 表
Read 10_事实源/desktop-state.md  # 找 Tauri command 表
Edit 加新 command

# 步骤 5: 验收
Read 20_工作项/.../验收.md
Edit 加一行: ✅ 缺口 NN | 完成日期 | 实际行数 | commit hash | 借鉴源

# 步骤 6: commit
git add -A
git commit -m "feat(core): 缺口 NN <名称> (<借鉴源> 同源)

- <改动 1>
- <改动 2>
- 测试: 248 + N passed
- 借鉴: <项目> <链接>

Refs: docs/设计/NN_*.md"

# 步骤 7: 下一个缺口
```

**关键纪律**:
- **不跳步骤** (6 步全做, 缺一不可)
- **不混 commit** (单缺口 1 commit, 集成点 1 commit, 不打包)
- **不省略测试** (每缺口至少 3 个新测试)
- **不省略文档** (runtime-state / desktop-state 必同步)
- **不写未要求的功能** (CLAUDE.md §4)
