# qx 交互式 TUI 实现记录

## 2026-06-01 P0 首轮

已完成:

- 将 `qianxun/src/tui/mod.rs` 从手写 ANSI raw mode 原型替换为 `ratatui` 全屏 TUI.
- 独立 CLI 入口改为 `main.rs -> cli::run::run_repl(config, project_root, global_instructions) -> tui::run(...)`.
- TUI 首屏包含 header, 对话区, 输入框, status 四区布局.
- 输入框使用 `tui-input`, 支持基础编辑和 Unicode 显示宽度游标.
- 支持 `/` 命令面板, 过滤, 上下选择, Tab/Enter 补全.
- 支持 `/help`, `/quit`, `/exit`, `/reset`, `/usage`, `/workspace`, `/skills`, `/tools`, `/memory`, `/retry`, `/edit`, `/mode`, `/plan`.
- 新增 `TuiOutputSink`, 将 `OutputSink` 事件转换成 TUI 内部事件.
- 接入真实 `processing_loop::handle_user_message`, 支持 LLM 流式文本, 工具调用卡片, token/status/error/turn finished 展示.
- 保留 `Mode::Auto` / `Mode::Plan` 工具过滤.
- 初始化 builtin tools, skill_read, MemoryCore, workspace MCP.
- 新增 TUI 最小单测: 命令过滤和 80x24 基础布局渲染.

验证:

- `cargo check -p qianxun`: 通过, 仍有既有 warning.
- `cargo test -p qianxun tui::tests -- --nocapture`: 通过, 2 个测试通过.

已知缺口:

- 会话恢复/列表/删除尚未迁入新 TUI.
- 历史输入上下翻尚未实现.
- 多行输入尚未实现.
- 工具权限审批弹窗尚未实现.
- 工具结果详情仍缺少 core 层 `OutputSink::on_tool_result` 事件.
- 旧 `qianxun/src/cli/cli.rs` 和 `qianxun/src/cli/output.rs` 已变成遗留路径, 当前编译会产生大量未使用 warning.
